信息安全面临的威胁与攻击类型

一、信息安全威胁四大分类

信息安全威胁总体分为四大类：

  • 人为蓄意破坏
  • 灾害性攻击
  • 系统故障
  • 人员无意识行为

二、人为蓄意破坏

分为 被动攻击 和 主动攻击。

1）被动攻击

特点：只窃取、不修改、不破坏，隐蔽性强，破坏保密性。

1. 网络监听（窃听）

攻击者通过抓包、链路监听等方式捕获网络传输数据，窃取账号、密码、隐私信息、业务明文数据。仅获取信息，不对数据做任何修改。

2. 业务流分析

不破解数据内容，通过长期统计通信流量、通信频率、数据包大小、收发时间、访问规律，推测业务结构、用户行为、重要节点，获取涉密信息。

3. 非法登录

攻击者尝试盗用账号密码试探登录系统，仅做访问探测，不篡改、不破坏数据，属于低危害、隐蔽式探测行为。

2）主动攻击

特点：篡改、伪造、中断服务，破坏完整性、可用性、真实性，可被日志记录。

通用主动攻击

1. 数据篡改

拦截网络传输的数据报文，修改其中的业务参数、内容信息后重新发送，使接收方获取虚假数据，破坏数据完整性。

2. 假冒身份

攻击者伪造合法用户身份、登录令牌、权限信息，冒充普通用户或管理员进入系统，越权访问、操作数据，破坏真实性。

3. 抵赖

用户或攻击者在操作完成后否认自己的行为。包括发送方否认发送消息、接收方否认接收消息，事后拒绝承担责任，造成行为无法追溯。

4. 旁路控制（旁路攻击）

不直接破解加密算法，通过观察设备运行时间、功耗、电磁辐射、噪音等旁侧特征，利用统计学分析反向推导密钥，破解加密系统。

5. 重放攻击

攻击者截取合法的数据包，在后续时间重复发送，重复完成登录、转账、授权等有效操作。通常通过 时间戳、随机数 防御。

6. 拒绝服务攻击（DOS）

向服务器发送大量无效请求、畸形数据包，耗尽服务器带宽、CPU、内存资源，导致正常用户无法访问服务，破坏可用性。

Web 专项主动攻击

1. XSS 跨站脚本攻击

网站未过滤用户输入内容，攻击者注入恶意 JS 脚本。用户访问页面时脚本自动执行，可窃取 Cookie、劫持会话、钓鱼、跳转恶意网站。

2. CSRF 跨站请求伪造

利用浏览器自动携带登录凭证的特性，诱导已登录用户点击恶意链接，冒用用户身份执行转账、改密、修改资料等非自愿操作。

3. 缓冲区溢出攻击

程序未校验输入长度，输入数据超出缓冲区容量，覆盖系统内存，攻击者可借此执行恶意代码、获取服务器最高权限。

4. SQL 注入攻击

网站直接拼接用户输入构造 SQL 语句，攻击者插入恶意 SQL 字符，欺骗数据库执行非法查询、删除、篡改数据。

  • 攻击方式：语句拼接、注释绕过、附加条件、非法传参
  • 防御方式：参数化查询、输入过滤、正则校验、预编译 SQL、漏洞扫描、数据加密

三、灾害性攻击（非人为威胁）

属于不可抗力物理灾害，直接造成设备损毁、机房瘫痪、数据丢失。

  • 水灾：机房进水导致设备短路、硬件损坏
  • 火灾：线路起火烧毁服务器、存储设备
  • 地震：破坏机房建筑与设备结构
  • 雷击：高压电流击穿网络设备、主板芯片
  • 战争：破坏网络基础设施、通信骨干网

四、系统故障（设备与环境故障）

无攻击者参与，属于基础设施自身故障。

  • 硬件故障：硬盘、内存、主板、网卡损坏导致设备宕机
  • 软件故障：系统崩溃、程序Bug、进程卡死、版本异常
  • 链路故障：光纤、网线、交换机端口故障导致网络中断
  • 供电故障：断电、电压不稳、UPS 失效导致设备异常关机

五、人员无意识行为（内部无意风险）

员工无恶意，但操作失误造成信息安全隐患。

  • 编程错误：代码漏洞、未过滤、未脱敏、权限漏洞
  • 操作错误：误删数据、误格式化、误开放端口、误操作配置
  • 无意泄密：电脑未锁屏、私传文件、误发涉密资料、随意丢弃纸质文件

六、安全属性对应总结

  • 被动攻击 → 破坏保密性（偷看、窃取）
  • 数据篡改 → 破坏完整性（改掉数据）
  • 假冒、伪造 → 破坏真实性（冒充身份）
  • 拒绝服务 → 破坏可用性（服务瘫痪）
